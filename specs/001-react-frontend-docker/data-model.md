# Data Model: React Frontend for Jogo da Velha

**Feature**: 001-react-frontend-docker  
**Date**: 2026-04-22

## Overview

The frontend does not own any persistent data. All entities below are **read-only representations** of backend state, received via the API contract. The TypeScript interfaces mirror the constitution's Game Object Schema exactly.

## Entities

### Game

Represents a Tic-Tac-Toe match. This is the single data structure returned by all 4 API endpoints.

| Field           | Type                        | Description                                      | Values                                     |
|-----------------|-----------------------------|--------------------------------------------------|--------------------------------------------|
| `gameId`        | `string`                    | Unique identifier generated by backend           | Opaque string (e.g., `"abc123"`)           |
| `board`         | `string[3][3]`              | 3×3 grid of cell values                          | Each cell: `""`, `"X"`, or `"O"`           |
| `currentPlayer` | `string`                    | Marker of the player whose turn it is            | `"X"` or `"O"`                             |
| `status`        | `string`                    | Current game lifecycle state                     | `"waiting"`, `"in_progress"`, `"finished"` |
| `winner`        | `string \| null`            | Winner marker, draw indicator, or null           | `"X"`, `"O"`, `"draw"`, or `null`          |

### ApiError

Represents an error response from the backend.

| Field   | Type     | Description              | Example Values                                     |
|---------|----------|--------------------------|----------------------------------------------------|
| `error` | `string` | Human-readable error msg | `"Game not found"`, `"Not your turn"`, `"Cell already occupied"`, `"Game is already full"`, `"Game is already over"` |

### MoveRequest

Represents the request body for making a move.

| Field    | Type     | Description                  | Values            |
|----------|----------|------------------------------|-------------------|
| `player` | `string` | Marker of the player moving  | `"X"` or `"O"`   |
| `row`    | `number` | Row index of the target cell | `0`, `1`, or `2`  |
| `col`    | `number` | Column index of target cell  | `0`, `1`, or `2`  |

## State Transitions

### Game Status Flow

```
waiting → in_progress → finished
```

| Current Status  | Trigger                          | Next Status    |
|-----------------|----------------------------------|----------------|
| `waiting`       | Second player joins (POST /join) | `in_progress`  |
| `in_progress`   | Three in a row achieved          | `finished`     |
| `in_progress`   | All 9 cells filled (no winner)   | `finished`     |

### Frontend View States

The frontend UI maps game status to view states:

| Game Status    | View State       | UI Behavior                                                    |
|----------------|------------------|----------------------------------------------------------------|
| (no game)      | `idle`           | Show GameControls (Create/Join). No board visible.             |
| `waiting`      | `waiting`        | Show board (empty). Show game ID to share. Poll for join.      |
| `in_progress`  | `playing`        | Show board with markers. Show turn indicator. Poll for updates.|
| `finished`     | `game_over`      | Show final board. Show winner/draw message. Board disabled.    |

## Relationships

```
Game 1 ──── has ────► Board (3×3 grid of cells)
Game 1 ──── has ────► 2 Players (X, O — identified by marker only)
MoveRequest ──► targets one Cell in the Board
```

## Validation Rules

All validation is performed by the backend. The frontend enforces only **UI-level guards**:

| Rule                        | Enforced By | Behavior                                      |
|-----------------------------|-------------|-----------------------------------------------|
| Cell must be empty to click | Frontend    | Disable/ignore click on occupied cells         |
| Must be player's turn       | Frontend    | Disable board when not the player's turn       |
| Game must not be finished   | Frontend    | Disable all cells when status is `"finished"`  |
| Valid row/col range (0–2)   | Frontend    | Derived from board grid — no manual input      |
| Turn order, win detection   | Backend     | Frontend defers to backend response            |
